# Solo Leveling System - 架构设计

> AI 对话驱动，三层数据模型，MCP/Skill 交互

---

## 三层模型：动机 → 目标 → 任务

```
┌─────────────────────────────────────────────────────────────┐
│  动机 (Motivation)                                          │
│  ├── 本质：人的底层驱动力，很少变                            │
│  ├── 数量：3-5个，不会太多                                  │
│  ├── 例子："我想被她喜欢"、"我想财务自由"                    │
│  └── 变化频率：几个月甚至几年才变一次                        │
├─────────────────────────────────────────────────────────────┤
│  目标 (Goal)                                                │
│  ├── 本质：动机的具体化，可衡量、有截止时间                  │
│  ├── 数量：每个动机下 1-3 个目标                            │
│  ├── 例子："6月见面时体脂15%"、"3月前月入2万稳定"           │
│  └── 变化频率：每周到每月可能调整                            │
├─────────────────────────────────────────────────────────────┤
│  任务 (Task)                                                │
│  ├── 本质：推进目标的具体行动                                │
│  ├── 数量：每天 3-10 个                                     │
│  ├── 例子："跳绳2000个"、"写代码4小时"                      │
│  └── 变化频率：每天都在变                                    │
└─────────────────────────────────────────────────────────────┘
```

## 数据关系

```
Motivation (动机)
├── id: "m1"
├── name: "被她喜欢"
├── why: "网恋对象明年见面，我想让她心动"
└── goals: [Goal]

Goal (目标)
├── id: "g1"
├── motivation_id: "m1"
├── name: "见面时体脂15%"
├── metric: "体脂率"
├── current: 25
├── target: 15
├── unit: "%"
├── deadline: "2025-06-01"
├── progress: 0.4  // 自动计算: (25-15)/(25-15) 的反向
└── tasks: [Task]

Task (任务)
├── id: "t1"
├── goal_id: "g1"
├── name: "晚间跳绳 2000个"
├── status: "pending" | "done" | "skipped"
├── date: "2025-12-02"
├── contribution: "体脂 -0.02%"  // 预估贡献
└── note: "..."
```

## AI 交互模式

AI 不是任务管理器，而是**对话伙伴**。

### 对话场景

```
用户: "我今天不太想动"

AI:
├── 1. 理解上下文 → 调用 MCP 获取今日任务
├── 2. 识别意图 → 用户在犹豫/拖延
├── 3. 关联动机 → 查看这个任务属于哪个目标/动机
├── 4. 回应 → "你离见她还有183天，现在体脂25%，目标15%。
│              每天跳绳能降0.02%，跳还是不跳？"
└── 5. 等待决策 → 用户说跳/不跳，更新任务状态
```

### AI 的角色

- **不是**：提醒机器人、任务管理器、打卡工具
- **是**：懂你动机的对话伙伴，帮你在犹豫时看清本质

## MCP / Skill 设计

### MCP Tools（数据操作）

```typescript
// 动机相关
mcp_motivation_list()        // 列出所有动机
mcp_motivation_get(id)       // 获取单个动机详情
mcp_motivation_create(data)  // 创建动机
mcp_motivation_update(id, data)  // 更新动机

// 目标相关
mcp_goal_list(motivation_id?)    // 列出目标
mcp_goal_get(id)                 // 获取目标详情（含进度）
mcp_goal_create(data)            // 创建目标
mcp_goal_update(id, data)        // 更新目标
mcp_goal_progress(id)            // 计算目标进度

// 任务相关
mcp_task_list(date?, goal_id?)   // 列出任务
mcp_task_create(data)            // 创建任务
mcp_task_done(id)                // 标记完成
mcp_task_skip(id, reason?)       // 跳过任务
```

### Skills（复合操作）

```typescript
/today          // 显示今日概览：动机进度 + 今日任务
/review         // 周/月复盘：目标进度、完成率、调整建议
/add-task       // 对话式添加任务，自动关联目标
/update-goal    // 更新目标状态（当前值、截止时间等）
/motivation     // 查看/编辑动机
```

## 交互流程示例

### 场景1：每日开始

```
用户: 打开软件 / 说 "今天干嘛"

AI 调用: mcp_goal_list() + mcp_task_list(today)

AI 输出:
┌────────────────────────────────────────┐
│ 今天是 12月2日，距离见她还有 183 天     │
│                                        │
│ 【被她喜欢】体脂 25% → 15%             │
│  ██████░░░░░░░░ 40%                    │
│                                        │
│ 【财务安全】月入 2万稳定               │
│  ███░░░░░░░░░░░ 20%                    │
│                                        │
│ 今日任务：                             │
│ ○ 游戏脚本开发 8小时                   │
│ ○ 晚间跳绳 2000个                      │
│ ○ 核心训练 15分钟                      │
└────────────────────────────────────────┘
```

### 场景2：完成任务

```
用户: "跳绳完了"

AI 调用: mcp_task_done("t1")

AI 输出:
"跳绳 2000 ✓
 预计体脂 -0.02%，累计本周 -0.1%
 按这个节奏，4月中能到15%。"
```

### 场景3：想放弃

```
用户: "今天不想练了，太累了"

AI 调用: mcp_goal_get("g1") // 获取相关目标

AI 输出:
"现在体脂25%，目标15%，还差10个点。
 剩余183天，平均每天要降0.055%。
 今天不练，明天要多降0.057%。

 累了可以减量，跳1000个？还是今天彻底休息？"

用户: "跳1000吧"

AI 调用: mcp_task_update("t1", {name: "跳绳1000个"}) + mcp_task_done("t1")
```

### 场景4：添加新目标

```
用户: "我想学 AI，感觉这个很重要"

AI: "学 AI 是为了什么？
     - 赚更多钱？
     - 做出自己的产品？
     - 还是单纯好奇？"

用户: "做产品吧，一人公司那种"

AI 调用:
  mcp_motivation_get("m3")  // 假设已有"做产品"动机
  mcp_goal_create({
    motivation_id: "m3",
    name: "掌握 AI 应用开发",
    metric: "能独立做 AI 产品",
    target: "完成一个可演示的 AI 产品",
    deadline: "2025-06-01"
  })

AI: "已添加目标：掌握 AI 应用开发
     截止：2025年6月
     关联动机：做出自己的产品

     要不要现在拆解第一步？"
```

## 数据存储

### 方案1：本地 SQLite（当前架构）

```
system-client/
├── electron/
│   └── db.ts          // SQLite 操作
└── src/
    └── ...            // React 前端
```

### 方案2：Markdown 文件（简单方案）

```
solo_leveling_system/
├── motivations.md     // 动机列表
├── goals.md           // 目标列表
├── 2025-12-02.md      // 每日任务
└── ...
```

AI 直接读写 Markdown，无需 MCP 后端。

### 建议

先用 **Markdown 方案** 验证核心交互，跑通了再迁移到 SQLite。

## 核心原则

1. **动机驱动** — 所有任务都要能回溯到动机
2. **对话优先** — AI 是对话伙伴，不是打卡机器
3. **进度真实** — 不要游戏数值，要真实的距离
4. **简单优先** — 先 Markdown，跑通再复杂化
