# Solo Leveling System v2 - 动机驱动设计

> 扔掉数值，回归本质：你想要什么？你离它有多远？

---

## 为什么扔掉数值？

数值是游戏的语言，不是人生的语言。

- "智慧 +3" — 然后呢？
- "经验值 2450/5000" — 升级了又怎样？
- "战斗力 28" — 这个数字代表什么？

**这些都是假反馈。**

真正驱动人行动的是：
> "我想要 X，我现在离 X 还有多远。"

---

## 核心模型：动机 → 目标 → 进度

```
┌─────────────────────────────────────────────────┐
│                                                 │
│   动机（WHY）                                   │
│   ↓                                             │
│   目标（WHAT）                                  │
│   ↓                                             │
│   进度（HOW FAR）                               │
│                                                 │
└─────────────────────────────────────────────────┘
```

### 示例

**动机：** 我想吸引她
**目标：** 2025年6月见面时，体脂15%，有腹肌
**进度：** 体脂25% → 15%，还差10个点，还有6个月

这比"美貌 35/100"有意义一万倍。

---

## 四大底层动机

人的所有行为，归根结底是这四种动机：

| 动机 | 本质 | 问自己的问题 |
|------|------|--------------|
| **避险** | 我不想遇到麻烦 | 什么事情可能毁掉我？ |
| **吸引** | 我想被需要、被爱 | 我想吸引谁？ |
| **积累** | 我想要安全感 | 多少钱我才能安心？ |
| **扩展** | 我想变得更强 | 我想能做到什么？ |

每个动机下面，是**具体的、可衡量的、有截止时间的目标**。

## 系统设计：只看两件事

### 1. 你的动机清单

不是任务清单，是**你真正想要的东西**。

```
我的动机清单
├── 【吸引】网恋对象见面时让她心动
```

## 任务的意义变了

传统系统：任务 → 经验值 → 升级
**新系统：任务 → 推进某个动机的进度**

每个任务不再是"值多少经验"，而是：

> 这个任务完成后，我离哪个目标更近了？近了多少？

```
任务：晚间跳绳 2000个
├── 推进动机：【吸引】见面时让她心动
├── 进度贡献：体脂 -0.05%（估算）
└── 累计效果：30天后体脂降低约1.5%
```

```
任务：完成游戏脚本开发
├── 推进动机：【积累】摆脱财务焦虑
├── 进度贡献：+3000元收入
└── 累计效果：存款增加，离目标更近
```

## 界面设计思路

不需要复杂的属性面板、进度条、等级系统。

**只需要一个画面：**

```
┌────────────────────────────────────────────┐
│                                            │
│  你现在最重要的三件事：                     │
│                                            │
│  1. 见面时让她心动                          │
│     ██████░░░░░░░░ 40%                     │
│     → 今天：跳绳 2000                       │
│                                            │
│  2. 摆脱财务焦虑                            │
│     ███░░░░░░░░░░░ 20%                     │
│     → 今天：游戏脚本 8小时                  │
│                                            │
│  3. 修复和爷爷的关系                        │
│     ░░░░░░░░░░░░░░ 0%                      │
│     → 12/4：回老家                          │
│                                            │
└────────────────────────────────────────────┘
```

**没有：**
- 智慧/美貌/金钱/战斗力
- 经验值/等级
- Buff/成就/金币

**只有：**
- 你想要什么
- 你离它有多远
- 今天该做什么

---

## 这样设计的好处

### 1. 真实
进度是真的。40%就是40%，不是游戏数值。

### 2. 聚焦
不是"我今天完成了5个任务"，而是"我离目标近了一点"。

### 3. 有意义
每个行动都能回答：这对我真正想要的东西有什么帮助？

### 4. 简单
不需要复杂的计算公式。目标明确，进度可见，就够了。

## 待解决的问题

### 1. 进度怎么算？

有些目标容易量化（体脂%），有些不容易（"让她心动"）。

**思路：** 拆解成可量化的子目标
- "让她心动" → 体脂15% + 穿搭升级 + 聊天能力提升
- 每个子目标有自己的进度，汇总成总进度

### 2. 动机会变吗？

会。人的动机会随时间变化。

**思路：** 定期复盘（每月），问自己：
- 这个还是我想要的吗？
- 有新的动机出现吗？
- 优先级变了吗？

### 3. 日常任务怎么和动机挂钩？

不是所有任务都能直接推进动机（比如"吃饭"）。

**思路：** 区分两类任务
- **推进型任务** — 直接推进某个动机
- **维护型任务** — 维持基本运转（可以不追踪）

---

## 总结

```
旧系统：任务 → 数值 → 虚假的成就感
新系统：动机 → 进度 → 真实的接近感
```

不要问"我今天获得了多少经验"。
要问"我离我想要的东西，更近了吗？"
