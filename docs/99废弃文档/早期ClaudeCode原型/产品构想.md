# AI 时代的「我独自升级」- 个人成长系统构想

## 灵感来源

《我独自升级》（Solo Leveling）- 韩国网络小说/漫画/动画

## 核心洞察

在当前 AI 时代，存在一个普遍问题：

- **AI 锁死了新手期发展** - 人们跳过了必要的练习和成长阶段
- **变得怠惰** - 过度依赖 AI，失去主动学习的动力
- **看不到进步方向** - 缺乏清晰的成长路径和目标
- **无法成为全能人** - 没有形成持续性挑战和突破的习惯

## 核心愿景

开发一个软件系统，实现：

1. **人与 AI 螺旋攀升** - 不是 AI 代劳，而是协同成长
2. **游戏化系统** - 类似游戏的升级机制，给人"打怪升级"的体验感
3. **可视化进步** - 让用户实打实地看到属于自己的成长轨迹

## 核心追问

**AI 给人提供的生活方式，可以显著改善一个人的生活质量吗？**

## 概念澄清

类比《我独自升级》：
- 主角变强 ← 背后有「系统」在引导
- 系统发布任务 → 主角完成 → 获得奖励/变强

**映射到现实：**
- AI 扮演「系统」的角色
- 根据用户情况，设计**生活作息相关的游戏性任务**
- 用户完成任务 → 可视化反馈 → 真实生活质量提升

**这不是抽象的"成长"，而是具体的生活指导：**
- 作息规律（几点睡、几点起）
- 日常习惯（运动、阅读、专注工作）
- 健康管理（饮食、休息、情绪）
- 技能练习（每日刻意练习）

## MVP 方案：基于 Claude Code + 文档系统

### 核心理念

**不需要开发 App，直接用对话 + 文档维护，就是一个可运行的"系统"。**

### 文档体系（AI 维护）

| 文档类型 | 作用 |
|---------|------|
| 每日计划 | 今天要完成的任务清单 |
| 当前时间安排 | 时间块规划 |
| 警示语 | 提醒自己的核心原则/戒律 |
| 焦虑待办思维导图 | 把焦虑外化成可处理的结构 |
| 任务拆分细节 | 大目标 → 小步骤 |
| Buff 库 | 获得的增益效果（知识、洞察、资源） |

### 交互方式

1. **用户对话输入：**
   - "我现在焦虑 XXX"
   - "我要完成 XXX，完成后能得到 XXX 钱"
   - "我刚看了一个很牛的视频，讲的是 XXX"

2. **AI 系统响应：**
   - 更新对应文档
   - 拆分任务
   - 触发/记录 Buff
   - 在后续场景智能提醒相关 Buff

### Buff 机制示例

**触发场景：** 用户看了一个战略投资人讲"一人公司"的视频

**系统记录：**
```
🔥 Buff 获得：一人公司战略视角
- 来源：[视频名称/链接]
- 核心洞察：XXXXX
- 触发条件：当讨论商业模式、公司规划时自动提醒
- 有效期：持续 / 或设定复习周期
```

**后续智能弹出：**
当用户聊到相关话题时，AI 主动提醒："你之前获得了「一人公司战略视角」Buff，里面提到 XXX，可能对当前决策有帮助。"

### AI 的核心角色：时间守护者

**人的弱点：**
- 对时间流逝不敏感，总觉得"还有时间"
- 容易被琐事淹没，忘记真正重要的事
- 说过的话、定过的目标，转头就忘
- 低估任务耗时，高估自己的执行力

**AI 的优势：**
- 精确感知时间：知道现在几点，距离 deadline 还有多久
- 记住用户说过的所有话：承诺、目标、焦虑、优先级
- 无情绪干扰：不会因为"不想做"而拖延提醒
- 全局视角：能看到用户看不到的时间冲突和机会成本

**系统行为：**
1. **基于当天对话，动态生成/调整日程**
   - 用户说"今天必须把 XX 搞定" → 自动排入高优先级
   - 用户说"下午有个会" → 自动避开这个时间段
   - 用户说"最近很焦虑 XX" → 安排时间专门处理

2. **主动提醒时间的珍贵性**
   - "现在是下午 3 点，你说今天要完成 XX，目前还没开始"
   - "这周已经过去 4 天了，XX 任务还在待办里"
   - "你上周说这个月要完成 XX，还剩 X 天"

3. **对抗用户的自我欺骗**
   - 用户说"待会再做" → 系统追问"具体几点？我帮你设提醒"
   - 用户说"这个不急" → 系统提醒"你上次也这么说，结果拖了两周"

### MVP 优势

- ✅ 零开发成本
- ✅ 立刻可用
- ✅ 数据全在本地，可控
- ✅ AI 天然理解上下文，比 App 更智能
- ✅ 可迭代：用着用着就知道需要什么功能
- ✅ AI 比用户更珍惜用户的时间
