# 主动询问与窗口记忆

> 状态：待设计

---

## 核心思路

Agent 当前的上下文全靠截屏后 AI 猜。但很多时候 AI 猜不到——同一个浏览器窗口可能是在摸鱼也可能是在查资料。**直接问主人比猜更准**。

询问结果存入「窗口-任务映射」记忆数据库，后续视频模态 AI 总结时作为强上下文注入，不再猜。

---

## 四个询问触发条件

### 1. 新窗口无映射 → 主动问

切换到一个还没有对应主线任务的窗口时，Agent 主动问：

> "你打开了 Figma，这是在做什么？"

主人按住 ⌘ 回答（语音或文字），Agent 做两件事：
- 如果是新主线 → 创建主线任务，关联该窗口
- 如果是已有主线的子步骤 → 在该主线下创建步骤，关联该窗口

### 2. 主人主动说明

主人随时可以主动告诉 Agent：

> "我现在切到 Chrome 是去查 Swift 并发的文档，给 SoloAgent 用的"

Agent 直接记录映射，不需要等触发。

### 3. 定时刷新（每 3 小时）

已有映射的窗口，每 3 小时重新问一次：

> "你现在在 VS Code 还是在推进 SoloAgent 开发吗？还是换了别的事？"

防止用户在同一个编辑器里已经切换了项目但 Agent 不知道。

### 4. 长时间不操作 → 问在别处做什么

idle 超过一定时间后，Agent 询问：

> "你离开电脑一会儿了，在做什么呢？"

可能的回答："去吃饭了"、"在看手机"、"在和人聊天"。
记录到时间线，填补空白时段。

---

## 窗口-任务映射数据库

```
记录的核心数据：

窗口特征（bundleId + windowTitle 模式）
    → 映射到哪个主线任务
    → 该主线下的当前步骤
    → 上次确认时间
    → 主人在这个窗口的习惯行为描述

不在电脑前时
    → 主人通常在做什么（吃饭/休息/手机/社交...）
    → 各种情况的频率和典型时段
```

这个数据库会越来越详细——Agent 逐渐建立起「什么窗口 = 什么事」的完整图谱，同时记住主人的生活规律。

---

## 注入视频模态 AI

这些记忆在 Phase 1/2 分析时作为**强上下文**注入 prompt，优先级高于 AI 自己的猜测：

```
当前 ContextAdvisor 输出（已有）：
  "活跃应用：VS Code，窗口标题含 SoloAgent"

新增「窗口记忆」注入：
  "【主人确认】VS Code + SoloAgent 项目 = 主线任务「开发独自升级系统」
   当前步骤：重构 BatchManager 的批次触发逻辑
   确认时间：30 分钟前

   主人习惯：在这个窗口通常连续工作 1-2 小时，
   中间会切到 Chrome 查文档（仍属同一主线），
   切到微信通常是短暂回消息（< 3 分钟）"
```

视频模态 AI 拿到这段上下文后，就不会再把「切到 Chrome 查 Swift 文档」误判为「浏览网页」。

## 与现有系统的关系

| 系统 | 接入方式 |
|------|---------|
| `ContextAdvisor` | `buildContextHint()` 新增窗口记忆段落，优先级最高 |
| `ShadowAgent` | 负责发起询问、解析回答、写入数据库 |
| `WindowMonitorService` | 窗口切换事件触发「新窗口无映射」检查 |
| `QuestEngine` | 从回答中创建/更新主线任务和步骤 |
| `VoiceService` | ⌘ 长按语音回答询问 |
| `PersistenceManager` | 窗口-任务映射持久化（SwiftData） |

## 持久化设计（草案）

```swift
// 窗口 → 任务映射
struct WindowQuestMapping: Codable {
    let bundleId: String
    let titlePattern: String       // 窗口标题关键词/正则
    var questId: String            // 关联的主线任务
    var currentStep: String?       // 当前步骤描述
    var habitDescription: String   // "通常连续工作 1-2h，中间查文档"
    var lastConfirmedAt: Date      // 上次主人确认时间
}

// 离开电脑时的习惯记录
struct AwayPattern: Codable {
    var description: String        // "吃午饭"
    var typicalTimeRange: String   // "12:00-13:00"
    var frequency: Int             // 出现次数
    var lastOccurrence: Date
}
```
