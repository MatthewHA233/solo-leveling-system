# 昼夜表 — 视觉风格

## 已选定：C. 电路走线 (Circuit Trace)

走线 = 活动，焊点 = 细目，并行走线 = 同时段多活动。
单元格半透明底色（活动分类颜色）+ 走线叠加 + 辉光。

### 为什么选电路走线

- 与"序号用线连起来"的需求天然契合
- 并列走线天然支持同一时段多活动
- 焊点/节点天然对应细目事件

---

## 单元格视觉设计

### 核心原则

- 不是纯色填充（太"Excel"）
- 一个类型事件一个颜色
- 信息密度是昼夜表的灵魂

### 截屏缩略图作为背景（已确认）

每个 5 分钟格子的背景 = 该时间点的截屏缩略图。
一眼就能看到一整天的屏幕变化流。

- 从 screenshots 目录加载对应时间段的截图，缩放到格子大小
- 迹线和焊点画在缩略图上方

### 事件显示方式

- 合并后的事件**显示事件标题文本**，文本跨越对应的区间单元格
- 不是只用颜色块表示，要能直接看到"在做什么"

### 细目事件（微步骤）

- 序号代表一个大事件完成过程中的微小步骤
- 序号之间用**线连起来**，表示这些微步骤属于同一个事件的推进过程
- 多条事件的序号线可以**并列在同一列单元格**中（多轨道）
- 视觉上能看到一个事件是怎样一步一步完成的

### 单元格并列事件

- 一个 5 分钟单元格**不是只能显示一个颜色**
- 同一时间段可能有多个并列事件（比如一边听音乐一边写代码）
- 单元格内用不同颜色按符合直觉的方式排布

### 当前时间指示器

- 不需要花哨的扫描线/三角标记
- 一条出现在对应列上的**短横线刻度**，简洁明了

## 候选方案（未采用，留作参考）

**A. 辉光线框（Neon Wireframe）**
- 单元格不填充，只用颜色细线框勾勒轮廓
- 活动密度越高，线框辉光越亮、线条越粗
- 参考：赛博朋克 HUD、Tron

**B. 点阵 LED（Dot Matrix）**
- 每个单元格是一组微型 LED 灯珠阵列（3×3 或 4×4）
- 不同事件点亮不同位置和颜色
- 参考：LED 广告牌、航天器仪表盘

**D. 等高线热力（Topographic Heatmap）**
- 等高线/地形图方式表达活动密度
- 美观但不适合表达离散事件
